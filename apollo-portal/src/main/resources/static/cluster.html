Tips:
  • By adding clusters,one same program may adopt different configuration in different clusters (for example, in different data centers)
  • If different clusters have adopeted the same configuration, it will be unnecessary to create clusters
  • Apollo will tacitly abstract from files/opt/settings/server.properties(linux)orC:\opt\settings\server.properties(windows)in the machine the idc attribute as the name of cluster , such as SHAJQ（Jinqiao Data Center）,SHAOY（Ouyang Data Center)
  • Cluster name created here will have to correspond with the idc attribute in the server.properties on the machine
(Deploy the cluster, for example:SHAJQ,SHAOY or customize the cluster, for example:SHAJQ-xx,SHAJQ-yy)
