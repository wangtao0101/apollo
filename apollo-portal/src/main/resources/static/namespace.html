Tips:
  • Applications may cover the configuration of public Namespace through relating to them
  • If applications are not needed to cover the configuration of public Namespace, then it will not be necessary to relate to public Namespace
  • Configuration of public Namespace could be read by any project
  • Through creating public Namespace, it could be achieved that the configuration of public components and the need of muiltiple applications to share one configuration
  • If other applications need to cover the configuration of the public part, they may relate to the public Namespace and then configurate those need covering in the related Namespace
  • If other applications do no need to cover the configuration of the public part, then it will not be necessary for them to relate to public Namespace
  • Configuration of private Namespace can only be obtained by its belonging appilications
  • Configuration by group management could be achieved through creating a private Namespace
  • Format for private Namespace could be xml、yml、yaml、json. You may obtain the content of Namespace whose format is not properties via the ConfigFile port of Apollo-client
