Gray Creation Notice

Through creating gray version, you may perform gray test to some configuration
The gray process is as listed:
  1.Create gray version
  2.Configurate gray configuration items
  3.Configurate gray rules. A private namespace can be gray processed according to the IP of Client while a public one can be processed according to the IP of both AppId and Client
  4.Gray release
Eventually, there are two possible results for the gray version:Full dose release and discarding the gray
Full dose release:The gray configuration should be combined with that of the major version and be released. All the Clients will adopt the combined configuration.
Discarding gray:Delete the gray version, and all the Clients will return to the configuration of the major version.
Attention:
  1.If the gray version has already been gray released, after modifying the gray rules, it will become effective immediately without requiring a second gray release.
